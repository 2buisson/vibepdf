### Cluster 1 — Visual & layout delta (2026-06-14)

- **D1.1 List-item anatomy = Two-line rows.** Every row is filename (primary) + a status caption beneath it. Caption shows: *checking* → "Checking…"; *valid* → "{N} pages"; *error-password* → "Password protected" (error color); *error-corrupt* → "Could not read file" (error color). Status is always visible (no hover-to-reveal). Trade-off accepted: taller rows, fewer visible at once. Error color comes from Fluent's semantic critical/caution text token — NOT the accent (accent reserved for Merge + selection).
- **D1.2 Sidebar width = Balanced ~320px** in the 900px default window (preview ~580px). Drag-resize clamps: **min ~240px, max ~50% of window width**. Width does not persist across launches (PRD §9.1).
- **D1.3 Empty/placeholder states = Text only.** "Add PDFs to get started" (empty list) and "Select a file to preview it" (empty preview) are muted, centered, **no glyph/illustration** — strict reading of PRD §10.
- **D1.4 [ASSUMPTION] Corner radius & control sizing = Fluent/WinUI defaults** (no custom radius). Override at Finalize if desired.
- **D1.5 [ASSUMPTION] Accent usage = Merge button fill + list selection highlight only** (carries PRD §10). Status/error coloring uses Fluent semantic tokens, never the accent.

### Cluster 2 — State patterns (2026-06-14)

- **D2.1 Checking state = text-only "Checking…"** in both the row caption and the preview pane — no ProgressRing/spinner. Extends the text-only ethos (D1.3) to the in-motion state. The per-file wall-clock guard still resolves a stuck check to *error-corrupt* so an item never stays permanently non-interactive (reconcile M1 rationale preserved; threshold tuned in architecture, legacy was 30s).
- **D2.2 Merge >2s progress = bar only**, thin bar above the action bar, no label. Strict silent-by-default (reconcile H3). Determinate when the engine reports per-file progress, else indeterminate (PRD assumption). Sub-2s merges show **nothing** (H3).
- **D2.3 Disabled-Merge tooltips = three distinct messages**, one per cause: empty/no-valid → "Add at least one PDF to merge"; flagged present → "Remove or replace files with errors"; still checking → "Wait for files to finish checking". Extends reconcile H2 (which required empty vs all-error) with the transient checking case.
- **Pinned (from PRD/reconcile, no change):** flagged preview attempts partial render + inline exclusion notice (L3); UI locks for execution phase only, preview stays scrollable; success `InfoBar` auto-dismiss ~8s + manual + Open folder, one-banner-max, Merge clears stale banner (M3); failure `InfoBar` specific-reason-preferred + manual-dismiss-only + no partial output + UI unlocks + list preserved; folder-missing → inline "Folder not found"; close-during-merge `ContentDialog` Keep merging (default) / Close anyway with cooperative cancel + temp cleanup.

### Cluster 3 — Voice & Tone / Microcopy (2026-06-14)

- **D3.1 Per-situation tone matrix restored (reconcile H1):** empty = warm invitation, single line; file error = factual, non-blaming; success = brief, affirming, one optional action; failure = honest, specific reason if available, **no apology filler**; disabled Merge = helpful, names the fix; overwrite = OS-owned (native Save dialog), app authors none.
- **D3.2 Canonical microcopy (locked strings):** empty list `Add PDFs to get started`; empty preview `Select a file to preview it`; checking `Checking…`; valid `{N} pages` / `1 page`; password `Password protected`; corrupt `Could not read file`; success `Merged successfully — {filename}` + `Open folder`; failure-with-reason `Not enough space on E:\.` / `Access denied` / `File not found: {name}`; open-folder fail `Folder not found`; buttons `Add PDF(s)` · `Merge` · `Move up` · `Move down` · `Remove` · `Open folder`. Disabled-Merge tooltips per D2.3.
- **D3.3 Generic merge-failure fallback = "Merge failed. Try again or check the files."** (fact + two remedies, no apology filler).
- **D3.4 Close-during-merge dialog:** title **"Stop the merge?"**, body **"A merge is in progress. Closing now cancels it and no file is saved."**, buttons `Keep merging` (default) / `Close anyway`.
- **D3.5 Flagged-file preview notice = generic exclusion (legacy L3):** "This file has an issue — it will be excluded from the merge." Specific reason is read from the row caption (D1.1).

### Cluster 4 — Interaction primitives (2026-06-14, all pinned/confirmed)

- **D4.1** Single-select; click selects + loads preview, kept in sync; clicking the selected row is a no-op; adding files never changes selection.
- **D4.2** Reorder via Move up/down on the selected file; selection follows; **preview does not reload/flicker** (M2); end-of-list disables; no drag-reorder.
- **D4.3** Add appends to bottom + **auto-scrolls to reveal new items** (L2); duplicates silently skipped; picker-cancel silent no-op (L4).
- **D4.4** Remove clears selection (no neighbor auto-select); preview → "Select a file to preview it".
- **D4.5** Pressing Merge dismisses any visible banner before the Save dialog opens (M3).
- **D4.6** Keyboard: accessibility descoped (PRD §5); keep WinUI native baseline only (ListView arrow nav, Tab order), no committed guarantees, no custom focus work.
